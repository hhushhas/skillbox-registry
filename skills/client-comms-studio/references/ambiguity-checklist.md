# Ambiguity Checklist

Use this when the client ask is still fuzzy.

## Lock these first

- what are they actually buying: advice, chatbot, integration, or full delivery
- what must be ready now: opinion, quote, proposal, demo, or implementation
- what is fixed: deadline, platform, language, stakeholder
- what is optional: add-ons, integrations, analytics, admin features

## Commercial questions

- setup fee vs recurring fee
- usage metric: conversations, seats, stores, tickets, or orders
- overage model
- support / maintenance expectations

## Technical questions

- standalone widget or in-product integration
- public-facing only or account/order-aware
- required platforms: Shopify, WhatsApp, web, mobile, CRM
- data/privacy constraints

## Response strategy

- if the user needs speed, give one recommended model and mark assumptions
- if risk is high, name the missing variable explicitly
- do not dump a taxonomy when one answer is good enough
