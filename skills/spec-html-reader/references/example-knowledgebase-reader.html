Locked Target Shape

The new domain is not a compatibility refactor. RAG names and public tool semantics are replaced.

domain

Ownership

  • KnowledgeBase is owned by `userId` only.
  • No `organizationId` participates in the first release.
  • Agents attach by `knowledgeBaseId`.

Storage Split

  • Raw/full extracted text: object storage.
  • Human-readable segments: Mongo.
  • Numeric vectors: Upstash Vector.

Provider Locks

  • Document embeddings: `voyage-4-large`.
  • Query embeddings: `voyage-4-lite`.
  • Dimensions: `1024`; metric: `cosine`.

Knowledge Tool Token

Tool calls are authorized by a signed, session-scoped token with Redis-backed active/revoked `jti` tracking.

security
KnowledgeToolTokenClaims
KnowledgeToolTokenClaims {
  sub: string; // userId
  agentId: string;
  knowledgeBaseId: string;
  indexName: string;
  dimensions: 1024;
  callPublicId?: string;
  sessionId?: string;
  scope: "knowledge:tool:search";
  aud: "knowledge-tool";
  jti: string;
  iat: number;
  exp: number;
}

Rules

  • Signing env: `KNOWLEDGE_TOOL_TOKEN_SECRET`.
  • TTL: `min(runtime session lifetime, 30 minutes)`.
  • Reusable within a session because one call/chat can search multiple times.
  • Reject expired, wrong audience/scope, revoked `jti`, missing session binding, disabled KnowledgeBase, or missing agent attachment.
Ultravox transport
selectedTool.authTokens.Authorization = "Bearer <knowledgeToolToken>"
temporaryTool.http.authHeaders = ["Authorization"]
POST /api/v2/knowledge/tool/search
body = { query: string }

Retrieval Lanes

Dashboard/debug gets evidence. Voice/tool gets capped context. The lanes deliberately diverge.

latency

Dashboard/debug lane

caller auth
  -> load KnowledgeBase by { _id, userId }
  -> embed query with voyage-4-lite
  -> query dedicated Upstash index
  -> hydrate KnowledgeSegment + KnowledgeDocument from Mongo
  -> return RetrievalEvidence[]

Voice/tool lane

Ultravox/tool call
  -> validate signed/session-scoped Knowledge tool token
  -> resolve knowledgeBaseId, userId, indexName, dimensions
  -> embed query with voyage-4-lite
  -> query Upstash with includeData/includeMetadata
  -> build capped answerContext from data.context
  -> return KnowledgeToolResponse
